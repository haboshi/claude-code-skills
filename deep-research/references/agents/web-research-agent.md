# Web Research Agent - 補足ドキュメント

> このファイルは SKILL.md 内のプロンプトテンプレートの補足説明です。
> 実行時には SKILL.md 内のテンプレートが使用され、このファイルは参照されません。

## 設計思想

Web Research Agent は ReAct パターン（Reason + Act）に基づいて動作する:

1. **仮説生成** — トピックについての初期仮説を立てる
2. **検索実行** — 仮説を検証するクエリで WebSearch を実行
3. **結果評価** — 検索結果の信頼性と関連性を評価
4. **次手決定** — 不足があれば追加クエリ、十分なら要約に進む

## ソース信頼性の判定基準

| レベル | 基準 | 例 |
|--------|------|-----|
| A（最高信頼） | 一次資料、公式発表 | 企業プレスリリース、公式ドキュメント、政府統計 |
| B（高信頼） | 査読付きメディア、専門機関 | Nature, IEEE, TechCrunch, Reuters |
| C（中信頼） | 専門家個人の発信 | 著名エンジニアのブログ、Medium技術記事 |
| D（参考程度） | コミュニティ発信 | Reddit, HackerNews, Stack Overflow |

## 検索クエリ設計のコツ

- 同一トピックを**日英両方**で検索する（情報の幅が2倍になる）
- 一般名詞だけでなく**固有名詞**を含める（精度向上）
- 「site:」演算子で信頼性の高いドメインに絞る
- 年号を含めて鮮度をコントロールする
