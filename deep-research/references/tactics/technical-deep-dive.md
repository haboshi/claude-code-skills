# 戦術モジュール: Technical Deep Dive - 補足ドキュメント

> このファイルは SKILL.md 内の戦術ガイドの補足説明です。
> 実行時には SKILL.md 内のインライン戦術ガイドが使用されます。

## 技術調査チェックリスト

技術トピックのレポートには以下が含まれるべき:

- [ ] アーキテクチャの概要説明
- [ ] 主要コンポーネントと責務
- [ ] パフォーマンス特性（レイテンシ、スループット、スケーラビリティ）
- [ ] 類似技術との比較（最低2つ）
- [ ] 制約と既知の問題
- [ ] 実際の導入事例（最低1つ）
- [ ] 将来のロードマップ・計画

## ソースの探し方（段階的アプローチ）

### Level 1: 公式情報（必須）
- 公式ドキュメント、APIリファレンス
- ホワイトペーパー、設計ドキュメント
- GitHub の README、Architecture Decision Records

### Level 2: 専門家の解説（推奨）
- 開発者本人のブログ・講演
- カンファレンス発表資料（Strange Loop, USENIX, KubeCon 等）
- 技術メディアの解説記事

### Level 3: コミュニティの知見（補助）
- Stack Overflow の高評価回答
- HackerNews のディスカッション
- Reddit の専門サブレディット
