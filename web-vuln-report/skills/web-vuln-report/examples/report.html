本診断では、認可された検証環境に対し非破壊手法で 14 件の指摘事項を検出した。最も優先度が高いのは 機微ファイル（.git／.env）の公開（VWR-001, CVSS 7.5）であり、ソースコードや認証情報の漏洩に直結しうるため即時是正を要する。次いで、入力値の無害化されない反射（反射型 XSS の兆候）、オープンリダイレクト、危険な HTTP メソッドの有効化が、フィッシングやセッション窃取の起点となりうる。

改善ロードマップ ／ 優先度

  1. 即時（〜1週間）: .git／.env 等の機微ファイルを公開領域から除去し、Web サーバで拒否設定。漏洩した可能性のある認証情報はローテーションする。
  2. 短期（〜1か月）: 反射型 XSS・オープンリダイレクトの入力/出力検証を実装。不要な HTTP メソッド（TRACE/PUT/DELETE）を無効化。
  3. 中期（〜3か月）: セキュリティヘッダ（CSP を Report-Only から段階導入、X-Content-Type-Options、X-Frame-Options、Referrer-Policy）と Cookie 属性（Secure/HttpOnly/SameSite）を全面適用。クライアントライブラリを最新化し SCA を CI に組み込む。
  4. 継続: 是正後の再診断と、リリースパイプラインへの自動セキュリティチェック統合。

診断手法と評価基準（準拠標準）

本診断は次の標準に準拠する。診断手法は OWASP WSTG（Web Security Testing Guide、 非破壊で機械的に検証可能な項目に限定）に基づき、検出事項を OWASP Top 10:2025（現行 公開版）と CWE で分類、深刻度は CVSS 4.0 基本値（CVSS-B）で評価し、検証要件は OWASP ASVS 5.0.0（該当する要件番号・レベルを所見に併記）を参照する。診断は認可された 範囲に限定し、データ改変・サービス妨害・破壊的操作を伴わない非破壊手法で実施している。

補足: (1) CVSS 4.0 のスコアは CVSS 3.1 と互換性がない（同一事象でも数値が異なる）。 本報告書は Base のみ（CVSS-B、脅威情勢 E は最悪ケース前提）を用いる。 (2) 一部のチェック（CSP・ 一部セキュリティヘッダ・SRI・混在コンテンツ）は WSTG v4.2 に専用テスト ID が無く、独自根拠で診断 している。 (3) CWE は本スキル独自付与（ASVS 5.0 は CWE 公式マッピングを廃止）。 (4) 自動・非破壊診断 は ASVS の技術検証の一部であり、ASVS 認証・OWASP トラストマークを意味しない。

制約事項・免責

  • 本診断は非破壊の観測・安全な能動プローブに基づく。認証後領域や業務ロジックの詳細な検査 （手動ペネトレーションテスト）は範囲外である。
  • 「古いコンポーネント」等の CVE 級判定は、外部脆弱性データベース（nuclei 等）を併用した項目に限る。 内蔵チェックのみの結果は構成・衛生面の指摘であり、個別 CVE の成立を保証するものではない。
  • 検出事項は診断時点のスナップショットであり、以降の変更は反映されない。
